# شرح الميزات التفصيلي - Features

## 1. نظام المصادقة

### تسجيل الدخول بالبريد/كلمة المرور
- تسجيل حساب جديد مع الاسم والبريد وكلمة المرور
- تشفير كلمة المرور بـ bcrypt (12 rounds)
- جلسات مُخزّنة في PostgreSQL (صالحة 7 أيام)
- كوكيز آمنة (`httpOnly`, `secure` في الإنتاج)

### تسجيل الدخول الاجتماعي
- **Google OAuth 2.0** - تسجيل دخول بحساب Google
- **Facebook OAuth** - تسجيل دخول بحساب Facebook
- **Apple Sign-In** - تسجيل دخول بحساب Apple
- ربط تلقائي: إذا كان البريد موجودًا يتم ربط الحساب الاجتماعي بالحساب الحالي

### التحقق من البريد
- يُرسل رابط تحقق عند التسجيل (صالح 24 ساعة)
- بانر تنبيه في لوحة التحكم لمن لم يتحققوا
- زر إعادة إرسال رابط التحقق

### إعادة تعيين كلمة المرور
- إرسال رابط إعادة التعيين بالبريد (صالح ساعة واحدة)
- التحقق من صلاحية الرابط قبل عرض النموذج
- تأكيد كلمة المرور الجديدة

## 2. لوحة التحكم (Dashboard)

### الإحصائيات
- **إجمالي العملاء** - عدد العملاء المسجلين
- **العقود النشطة** - عدد العقود بحالة "نشط"
- **الفواتير المعلّقة** - عدد ومجموع الفواتير غير المدفوعة
- **المشاريع النشطة** - عدد المشاريع قيد التنفيذ

### التنبيهات
- **فواتير متأخرة** - فواتير تجاوزت تاريخ الاستحقاق (مع اسم العميل والمبلغ)

### بانر التحقق
- يظهر فقط للمستخدمين الذين لم يتحققوا من بريدهم
- قابل للإغلاق مؤقتًا

## 4. إدارة العقود

### إنشاء العقود
- **3 قوالب عربية جاهزة:**
  - عقد تقديم خدمات
  - عقد تنفيذ مشروع
  - عقد استشارات
- كل قالب يملأ العنوان والمحتوى تلقائيًا

### خصائص العقد
- **ربط بعميل** عبر قائمة منسدلة
- **القيمة والعملة** (SAR/USD/EUR)
- **تاريخ البداية والنهاية**
- **الحالة:** مسودة، نشط، مكتمل، منتهي، ملغى

### تصدير PDF
- تصدير العقد كملف PDF احترافي
- يتضمن هوية الشركة (الاسم، العنوان، الرقم الضريبي، الشعار)
- منطقة توقيع لكلا الطرفين
- تنسيق عربي RTL

## 5. إدارة الفواتير

### الترقيم التلقائي
- تنسيق `INV-XXXX` (مثل INV-0001, INV-0002)
- الرقم التالي يُحسب تلقائيًا من السيرفر

### بنود الفاتورة
- إضافة بنود متعددة (وصف، كمية، سعر الوحدة)
- حساب تلقائي لإجمالي كل بند
- إضافة وحذف بنود ديناميكيًا

### حساب الضريبة
- **نسبة VAT افتراضية:** 15% (معيار المملكة العربية السعودية)
- النسبة قابلة للتعديل
- حساب تلقائي: المجموع الفرعي → ضريبة → الإجمالي

### الحالات
| الحالة | الوصف |
|--------|-------|
| مسودة (draft) | فاتورة قيد الإعداد |
| مُرسلة (sent) | تم إرسالها للعميل |
| مدفوعة (paid) | تم السداد |
| متأخرة (overdue) | تجاوزت تاريخ الاستحقاق |
| ملغاة (cancelled) | تم إلغاؤها |

### طرق الدفع
- تحويل بنكي
- نقدي
- إلكتروني

### تصدير PDF
- فاتورة احترافية بتنسيق عربي
- جدول بنود مع تلوين متناوب
- تفاصيل الشركة والعميل
- ملخص المبالغ والضريبة

## 6. إدارة المشاريع والمهام

### المشاريع
- **ربط بعميل** و/أو **عقد**
- **الأولويات:** منخفض، متوسط، عالي، عاجل (مع تلوين)
- **الحالات:** لم يبدأ، قيد التنفيذ، معلّق، مكتمل
- **تاريخ بداية وموعد نهائي**
- **ميزانية** بالريال

### لوحة كانبان للمهام
عند فتح مشروع، تظهر لوحة بـ 4 أعمدة:

| العمود | الحالة |
|--------|--------|
| قيد الانتظار | todo |
| قيد التنفيذ | in_progress |
| مراجعة | review |
| مكتمل | done |

- كل مهمة تعرض: العنوان، الوصف، الأولوية
- تغيير حالة المهمة مباشرة عبر قائمة منسدلة
- **شريط التقدم** يحسب النسبة المئوية تلقائيًا

## 7. المستندات والتوقيع الإلكتروني (شبيه PandaDoc)

> توثيق تفصيلي في [DOCUMENT-SYSTEM.md](./DOCUMENT-SYSTEM.md)

### أنواع المستندات
1. **مستند نصي** — إنشاء محتوى عبر محرر TipTap الغني مع حقول قابلة للتعبئة
2. **مستند ملف** — رفع PDF/صورة ووضع حقول النماذج بصريًا

### محرر النصوص الغني (TipTap)
- **التنسيق:** غامق، مائل، تحتي، مشطوب، تمييز، كود
- **العناوين:** H1, H2, H3
- **القوائم:** نقطية، مرقمة
- **محاذاة:** يمين، وسط، يسار
- **عناصر متقدمة:** جداول، صور، روابط، اقتباسات، خطوط فاصلة
- **ألوان النص** مع قائمة منسدلة
- **حفظ تلقائي** كل ثانيتين مع مؤشر حالة مرئي

### الحقول القابلة للتعبئة (Fillable Fields)
إضافات TipTap مخصصة تظهر كشارات ملونة في المحرر وتتحول لحقول تفاعلية عند التوقيع:
- **حقل نص** — لإدخال نص حر (شارة زرقاء)
- **حقل تاريخ** — لاختيار تاريخ (شارة خضراء)
- **حقل توقيع** — لالتقاط التوقيع اليدوي (شارة بنفسجية)
- **حقل أحرف أولى** — للأحرف الأولى من الاسم (شارة برتقالية)

### المتغيرات الذكية (Smart Variables)
إدراج بيانات ديناميكية تُستبدل تلقائيًا عند التصدير:
- `{{profile.company}}` — اسم الشركة
- `{{profile.name}}` — الاسم الكامل
- `{{date.today}}` — التاريخ الحالي

### قوالب المستندات (15+ قالب)
- **عقود:** خدمات، مشروع، توظيف، اتفاقية سرية
- **مستندات:** عام، خطاب رسمي، محضر اجتماع
- **عروض:** سعر خدمات، مشروع، استشارات
- معاينة القالب قبل الاستخدام
- صفحة قوالب عامة على `/templates`

### مكتبة المحتوى (Content Library)
- حفظ محتوى متكرر لإعادة استخدامه
- تصنيف حسب النوع
- إدراج سريع من الشريط الجانبي

### محرر المستندات المرئي (PDF Editor)
- رفع PDF/صورة وعرضها كخلفية
- **وضع حقول النماذج بالنقر** على الموقع المطلوب
  - حقل نص، حقل تاريخ، حقل توقيع
- تعديل الأبعاد والموضع والتسمية لكل حقل
- خصائص: مطلوب/اختياري، عرض/ارتفاع، رقم الصفحة
- حفظ دفعي لجميع الحقول

### تدفق التوقيع الإلكتروني

```
1. إنشاء المستند ووضع الحقول
        ↓
2. إرسال للتوقيع (اختيار عميل أو إدخال بريد المستلم)
        ↓
3. توليد رابط مشترك فريد (shareToken)
        ↓
4. المستلم يفتح الرابط /sign/:token (بدون تسجيل دخول)
        ↓
5. المستلم يملأ الحقول ويرسم توقيعه
        ↓
6. يُحفظ: التوقيع + قيم الحقول + اسم الموقّع + البريد + IP + التاريخ
        ↓
7. ربط تلقائي بالعميل (إذا تطابق البريد)
        ↓
8. حالة المستند تتغير إلى "signed" + إشعار للمالك
```

### صفحة التوقيع العامة
- تعمل بدون مصادقة (رابط مشترك)
- عرض المستند مع الحقول القابلة للتعبئة
- لوحة رسم التوقيع (react-signature-canvas)
- التحقق من الحقول المطلوبة قبل الإرسال
- منع التوقيع المتكرر
- نص قانوني يوضح أن التوقيع الإلكتروني يعادل التوقيع اليدوي

## 12. لوحة الإدارة

### الوصول
- يتطلب دور `admin` أو `superadmin`
- رابط في الشريط الجانبي (يظهر فقط للمدراء)

### الإحصائيات
- إجمالي المستخدمين + المستخدمين الجدد اليوم
- المستخدمين النشطين + الجدد هذا الأسبوع
- المستخدمين المعلّقين
- إجمالي العقود
- إجمالي الملفات الشخصية والعملاء

### إدارة المستخدمين
- **بحث** بالاسم أو البريد
- **تعديل الدور:** User أو Admin (لا يمكن منح Superadmin من الواجهة)
- **تعديل الاشتراك:** تغيير الخطة والحالة يدويًا
- **تعليق/تفعيل** الحسابات
- **حذف** المستخدمين (مع حماية ضد حذف النفس أو Superadmin)

### أداة ترحيل البيانات
لترحيل البيانات من منصة Bubble.io السابقة:

1. **معاينة البيانات** - عرض عدد السجلات في كل ملف JSON
2. **ترحيل المستخدمين** - إنشاء حسابات بكلمات مرور عشوائية + إرسال بريد إعادة تعيين
3. **ترحيل العملاء** - ربط بالمستخدمين عبر خريطة المعرفات
4. **ترحيل العقود** - تحويل الحالات من العربية للإنجليزية
5. **ترحيل الملفات الشخصية** - مع أسماء مستخدمين تلقائية
6. **إعادة تعيين** - مسح حالة الترحيل للبدء من جديد

## 14. تصدير PDF

### الآلية
1. عرض HTML مخفي خارج الشاشة بالتنسيق المطلوب
2. تحويل إلى Canvas عبر `html2canvas` (دقة 2x)
3. تحويل Canvas إلى PDF عبر `jsPDF`

### يُستخدم في
- تصدير العقود
- تصدير الفواتير

## 15. رفع الملفات

### الآلية (Presigned URL)
1. العميل يرسل بيانات الملف للسيرفر
2. السيرفر يولّد Presigned URL من Object Storage
3. العميل يرفع مباشرة إلى Storage
4. العميل يحفظ المسار في قاعدة البيانات

### المكونات
- **useUpload Hook** - إدارة تدفق الرفع مع تتبع التقدم
- **ObjectUploader** - واجهة Uppy مع S3-compatible upload
- **FileUploadButton** - زر رفع بسيط مع معاينة

### الأماكن المُستخدمة
- صورة الملف الشخصي
- شعار الشركة
- صورة الغلاف
- صور الخدمات
- صور/فيديو المعرض
- مستندات PDF/صور

## 16. دعم اللغة العربية (RTL)

- **الاتجاه الأساسي:** RTL (من اليمين لليسار)
- **الخط:** IBM Plex Sans Arabic مع Inter كاحتياطي
- **القائمة الجانبية** على الجهة اليمنى
- **أيقونات التنقل** معكوسة (ChevronsRight = للصفحة الأولى)
- **رسائل الخطأ** بالعربية
- **تنسيق العملات:** `toLocaleString("ar-SA")`
- **محرر TipTap** يدعم الكتابة العربية و LTR معًا (`dir="auto"`)
- **تنسيقات CSS محددة لـ RTL:**
  - `padding-right` للقوائم (بدل `padding-left`)
  - `border-right` للاقتباسات (بدل `border-left`)
